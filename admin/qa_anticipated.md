## A. 측정과학 (8개) — 학계 평가위원 대비

### A1. "GUM은 정확히 어떻게 적용했나요?"
ISO/IEC Guide 98-3의 6개 불확도 성분(센서 선형성, 교정 기준 가스, 드리프트, 환경 노이즈, 디지털화, 환경 보정)을 numpy로 직접 구현했습니다. 합성표준불확도 u_c는 각 성분의 분산 합의 제곱근, 확장불확도 U는 k=2 적용으로 95% 신뢰구간을 산출합니다. 코드는 `src/innovation/gum_uncertainty.py`에 공개되어 있습니다.

### A2. "ISO 18516 1저자 기여라는데, 어떤 부분인가요?"
ISO 18516:2019 Annex A의 측정 프로토콜 섹션이 제 2017년 *Nanoscale* 논문 (DOI: 10.1039/C7NR00150B)을 기반으로 작성됐습니다. ISO/TC 229 WG 2에서 5년간 표준 제정에 참여했습니다.

### A3. "GUM 가중치(1/u²)가 normalization 없으면 학습 폭주 안 하나요?"
정확한 지적입니다. 본 구현은 weights = 1/(u² + ε)로 ε=1e-6 안정화 후 평균 1로 정규화합니다. 학습률 호환성과 수치 안정성을 동시에 확보합니다.

### A4. "Welch-Satterthwaite DoF가 코드에 있던데, 어디에 쓰이나요?"
Type A(통계적) + Type B(비통계적) 성분이 섞일 때 t-분포 보정 k 값 결정에 사용됩니다. 본 보고서에서는 k=2(95%) 가정이지만, DoF가 30 이하면 k 보정이 필요합니다. 함수는 구현돼 있고 분석 시 자동 적용됩니다.

### A5. "Cross-Source Calibration의 거리 보정 모델은 검증됐나요?"
현재 Gaussian Plume 단순화 모델 + decay_constant=0.15 (경험치)입니다. 5월 2일 실데이터 도착 후 회귀로 보정 예정입니다. 본 시뮬은 모델 구조 검증용이며, 실데이터 기반 정량 보정이 본 작업입니다.

### A6. "Page-Hinkley 외 다른 변화점 검출 알고리즘은 검토하셨나요?"
CUSUM, Bayesian Online Change Point Detection, ADWIN을 검토했습니다. Page-Hinkley는 (1) 매개변수 적음 (delta, threshold) (2) 점진 드리프트에 민감 (3) 환경 측정 시계열에 잘 맞습니다. ruptures 라이브러리로 추가 비교 가능합니다.

### A7. "측정 불확도 출력에 신뢰구간이 어떻게 전파되나요?"
세 가지 방법: (1) Heteroscedastic Gaussian NLL — 모델이 평균과 분산 동시 예측, (2) Monte Carlo dropout (Bayesian approximation), (3) MC uncertainty propagation — 입력 u_x로 모델을 100회 샘플링. 본 구현은 (3)을 기본, (1)을 옵션으로 제공합니다.

### A8. "측정과학과 데이터과학 융합의 다른 사례가 있나요?"
드물지만 있습니다. 의료 영상 분야에서 MRI 측정 노이즈를 ML loss에 통합한 사례, 분광학에서 SNR 가중 학습 등. 환경 측정 분야 + 운영 자동화 + 정책 임팩트까지 통합한 사례는 제가 알기로는 없습니다.

## B. ML 기술 (8개) — 데이터사이언스 평가위원 대비

### B1. "왜 LSTM-Autoencoder인가요? Transformer는?"
세 가지 이유: (1) TMS 5분 간격 30일 = 8,640 step으로 sequence 길이가 적당, (2) 환경 측정 시계열은 강한 주기성(일·주·계절) — RNN 계열이 inductive bias 유리, (3) 학습 데이터·계산 자원 제약. PatchTST 같은 Transformer는 Phase 2 (실데이터 충분 시) 검토 예정입니다.

### B2. "Foundation Model 미세조정은 검토하셨나요?"
Chronos, MOIRAI 등 시계열 Foundation Model을 검토했습니다. 단점은 (1) 측정 불확도 통합이 어렵고 (2) 환경 도메인 사전학습 데이터 부족. 본 솔루션의 Innovation A를 적용하면 일반 Foundation Model로는 못 하는 작업이 됩니다.

### B3. "PINN(Physics-Informed Neural Network)은 왜 안 썼나요?"
Arrhenius 연소식 + 화학 평형 통합을 검토했지만, (1) 16일 일정 내 검증 어려움, (2) 발전소 보일러 물리식의 영업비밀 부분 접근 불가. Tier 2의 Measurement-aware ML이 더 직접적인 차별화입니다. PINN은 후속 연구 가능.

### B4. "Isolation Forest의 contamination 5%는 어떻게 정했나요?"
환경 모니터링 분야의 일반적인 anomaly rate (3~7%) 중간값으로 시작했습니다. 5월 데이터 받은 후 ROC curve로 최적값을 결정합니다.

### B5. "SHAP TreeExplainer 외에 다른 XAI는?"
LIME, Permutation Importance, Integrated Gradients를 검토했습니다. SHAP은 (1) shapley value의 이론적 정합성, (2) sklearn·XGBoost·PyTorch 모두 지원, (3) 일관성 점수(consistency_score) 정의 가능. 본 솔루션은 SHAP을 메인, Permutation을 fallback으로 사용합니다.

### B6. "데이터 누수(leakage)는 어떻게 방지했나요?"
세 가지 안전 장치: (1) 시간 기반 train/val/test 분할 (random split 금지), (2) ETL에서 미래 정보 사용 금지 (시계열 forward fill만), (3) 평가지표는 hold-out test set 단독 산출. 모든 random seed 고정.

### B7. "5호기를 단일 모델로 학습 vs 호기별 fine-tune 어느 게 낫나요?"
3단계 배포 권장: Phase 1 단일 모델 (보편 패턴 학습), Phase 2 호기별 fine-tune (마지막 layer만 재학습), Phase 3 자체 학습 (전체 재학습). Federated Learning은 발전사 간 데이터 공유 없이 학습 가능 — 향후 검토.

### B8. "모델 성능이 가상 데이터 기준이라면 실데이터에서 떨어지지 않을까요?"
정확한 우려입니다. 본 발표 전 (6/19~6/22) 실데이터 기반 재계산 후 슬라이드 갱신 예정입니다. 다만 Innovation A·B·F는 데이터 종속이 적은 구조적 차별화라 영향 작습니다.

## C. 정책·운영 (6개) — 임원 평가위원 대비

### C1. "환경부가 정말 본 솔루션의 표준화를 채택할까요?"
3가지 근거: (1) ISO/IEC 17025 §7.8.3은 측정 불확도 보고를 이미 의무화 (한국 미적용), (2) EU CBAM 2026 시행으로 측정 정확도 압박 증가, (3) 한국이 ILAC-MRA 가입국으로 국제 호환성 필요. 본 솔루션은 환경부가 채택 안 해도 발전사 자체 도입으로 작동합니다.

### C2. "발전사 운영자가 본 솔루션을 받아들일까요?"
Operator Trust Calibration이 정확히 그 우려를 답합니다. 자동화 비율을 운영자가 조정 가능 (Auto-Execute 기준 70/80/90 등). 점진 적용 — 첫 6개월 P2까지만 자동, 그 후 P1 자동 확대 가능. 운영자 책임 범위가 명확해져 오히려 부담 감소.

### C3. "본 솔루션의 한계는 무엇인가요?"
3가지: (1) Gaussian Plume 단순화 모델 — 복잡 지형(산악)에서 정확도 저하 가능, (2) 비지도 이상탐지 라벨 검증 어려움, (3) 평가위원이 GUM을 이해하지 못하면 차별화 평가 불공정 가능. 1쪽 인포그래픽으로 마지막 위험 완화.

### C4. "EU CBAM 절감액 261억이 너무 큰 거 아닌가요?"
보수적 가정 기반입니다. 측정 정확도 5%→3% 개선, k=2(95%) 보수 마진 축소, EU €90/t. 만약 정확도 개선 1%p (보수)와 €60/t (낮은 가격)이면 87억으로 줄어듭니다. 가격 시나리오 €60~€150 모두 시뮬레이션 가능합니다.

### C5. "5호기 적용 정량 추정 근거는?"
(1) 연 발전량은 한국전력거래소 EPSIS 공개, (2) CO2 집약도는 IPCC 가이드라인, (3) tier_factor는 호기 규모·연료별 보정. 모든 가정과 코드는 GitHub에 공개되어 평가위원 직접 검증 가능합니다.

### C6. "한국서부발전이 본 솔루션을 도입할 경우 누가 사용·관리하나요?"
운영 매뉴얼에 1주 운영 절차 명시: 운영자 1명 일 30분, 자동 일·주·월·분기 리포트, 임계 자동 대응 정책. 도입 시 본 연구자가 6개월 무료 컨설팅 제공.

## D. 비용·운영 (4개)

### D1. "솔루션 도입 비용은 얼마나 드나요?"
1호기 기준: CAPEX 5억(라이선스+SI+인프라+교육+컨설팅), OPEX 연 1억. 5호기 확장 시 CAPEX 약 12억(규모의 경제), OPEX 연 4억.

### D2. "Payback Period는?"
1호기 기준 약 7일. CBAM 절감 261억 + 교정 절감 1.5억 + 운영자 시간 절감 0.3억 = 연 약 263억 vs 도입 5억. 회수 매우 빠름. 실제 ROI는 5년 누적 약 1,500억 (호기당).

### D3. "본 연구자 단독 개발 솔루션이 운영 안정성 있나요?"
GitHub Public + CI/CD 자동 테스트 + Streamlit Cloud 배포. 코드 1,800+ LOC, 셀프테스트 9개 모두 통과. End-to-end 통합 테스트 자동화. MetroAI v0.5.0 (8500 LOC, 69/69 테스트) 검증된 기반.

### D4. "유지보수는 누가 하나요?"
1차 도입 시 본 연구자가 컨설턴트로 6개월 동행. 그 후 발전사 내부 IT팀 + 분기 외주 (예산 1,000만원/년 가정). 또는 KEITI·KRISS 같은 정부기관 위탁 가능.

## E. 한계·약점 (4개) — 평가위원 함정 대비

### E1. "측정 정확도 1%p 개선이 진짜 가능한가요?"
보수적 추정입니다. 실제 발전사 데이터에서 검증 필요. 다만 GUM 가중 학습 + 캘리브레이션 드리프트 자동 탐지의 결합 효과는 단독 모델 적용 대비 정확도 1~3%p 개선 가능 (학술적 근거 — Bayesian deep learning 분야 다수 논문).

### E2. "본 솔루션이 너무 박사급 지향이라 일반 운영자에겐 어렵지 않나요?"
Operator Trust Calibration이 정확히 그 답입니다. GUM·Bayesian inference 같은 박사급 개념은 백엔드에서만 작동, 운영자는 0~100 점수 + P1/P2/P3 등급만 봅니다. 운영 매뉴얼은 일반 엔지니어 수준으로 작성됐습니다.

### E3. "평가위원이 GUM을 모르면 본 솔루션의 차별화가 평가받지 못하지 않나요?"
정확한 지적입니다. 1쪽 GUM 인포그래픽 (별첨 A)을 첨부했고, 본 발표에서도 슬라이드 6에서 직관적 도식으로 설명합니다. 또한 ISO 17025·17034·18516은 환경 평가위원이 들어봤을 표준이라 인용으로 신뢰 확보.

### E4. "실패 시 책임은 누가 지나요? 자동화 알람이 잘못되면?"
Operator Trust Calibration이 답입니다. 자동 실행(P1)은 OTS 80+ 시만, 즉 모델이 매우 확신할 때만 작동. 그 이하는 운영자 검토 후 실행. 책임 분담이 점수로 명확. 환경부 보고는 운영자 최종 승인 필수로 설계 가능.

## A 측정과학 추가 4 (9~12)

### A9. "태안화력 실측 데이터 (2014-08)는 너무 옛 데이터 아닌가요?"
정확한 지적입니다. 한국서부발전 자체 오픈 API의 sample data가 2014-08 한 시점입니다. 다만 본 시점 데이터로도 **8개 호기 (8611~8618) 호기 간 SOx 변동성 σ=11 PPM**을 추출했고, 이는 GUM-weighted loss 학습 데이터로 충분합니다. 운영 단계에서 한국서부발전과 데이터 활용 협약을 맺으면 최근 5분 간격 데이터 즉시 적용 가능합니다.

### A10. "GUM Type A 불확도와 Type B 불확도를 어떻게 구분하나요?"
ISO/IEC Guide 98-3 §4.2에 따라 Type A는 통계적 분석 (반복 측정 표준편차), Type B는 비통계적 (사양서·교정증명서·전문 판단). 본 구현은 TMS 측정에서 (a) 센서 반복성 = Type A, (b) 교정 기준 가스·드리프트·환경 노이즈·디지털화 = Type B로 분류 후 각각 분산 합으로 합성합니다. `src/innovation/gum_uncertainty.py:build_tms_uncertainty_budget()`에 구현됩니다.

### A11. "Heteroscedastic Gaussian NLL이 분산 underestimation 문제는 없나요?"
정확한 지적입니다. Kendall & Gal (2017) NeurIPS 논문에서 NLL이 high-variance 영역에 underfit 경향이 있다 보고됐습니다. 본 구현은 (1) σ_min clipping (σ < 0.01 차단), (2) Monte Carlo dropout 보조 추정, (3) MC propagation 100회 ensemble로 보완합니다.

### A12. "측정과학과 머신러닝 융합 연구의 학회 trend는?"
ISMQC (Int'l Symposium on Measurement Quality Control), CIM (Conf. on Metrology), MSC (Measurement Sci. Conf.) 등에서 점차 주제로 등장 중이지만, 환경 측정 분야 + 운영 자동화까지 통합한 사례는 본 연구가 사실상 첫 시도입니다. 향후 ISO/TC 229 WG와 협의하여 표준 부속서 개정 제안 검토 중입니다.

## B ML 기술 추가 4 (9~12)

### B9. "GARCH 변동성 모델은 검토하셨나요? (시계열 변동성 클러스터링)"
검토했고 Phase 2 후보입니다. 환경 측정 시계열은 (1) 일주기 + (2) 풍속·기온 시간 의존 변동성을 가집니다. GARCH(1,1)로 σ_t 모델링 후 GUM Type A에 통합 가능. 다만 16일 일정 내 검증 어려워 본 보고서에는 정적 GUM만 적용.

### B10. "Conformal Prediction은 검토했나요? Coverage Rate 보장에 더 적합할 텐데요."
적절한 지적입니다. Conformal Prediction은 distribution-free 90% coverage 보장이 강점이고, 본 §Ⅲ Coverage Rate 0.96 ✓ 결과와 직접 비교 가능합니다. 다만 Conformal은 점 예측 신뢰구간만 제공, 본 GUM-weighted loss는 학습 자체에 불확도 가중. 두 방법은 보완적이고 Phase 2 ensemble 검토 중입니다.

### B11. "Active Learning으로 라벨 비용 줄일 수 있을까요?"
환경 모니터링 분야 라벨 부재가 핵심 문제입니다. 본 구현은 Isolation Forest 비지도 학습으로 초기 ranking 후, OTS < 40 (수동 분석 권고) 알람만 운영자 라벨링 → 모델 재학습 cycle. 이는 본질적으로 uncertainty-based Active Learning. Phase 2에서 BALD·MC dropout 기반 명시적 AL 적용 검토.

### B12. "Federated Learning으로 발전사 간 데이터 공유 없이 학습 가능한가요?"
가능합니다. 5대 발전사 (한국남부·동서·서부·중부·남동) 각각 local 모델 학습 + Federated averaging으로 global 모델 구축. 데이터 주권 + 영업비밀 보호 동시 달성. 다만 발전사 간 합의 + 환경부 표준화 단계가 필요해 본 보고서는 단일 발전사 모델만 제시.

## C 정책·운영 추가 4 (7~10)

### C7. "환경부 공식 채택까지 절차와 일정은?"
3단계 절차: (1) 환경부 정책 제안 (본 보고서) → (2) 시범 사업 (한국서부발전 1호기, 6개월) → (3) ISO/IEC 17025 외부 인증 → 환경부 가이드라인 개정. 일반적으로 정책 채택 1~3년 소요. 본 솔루션은 자발적 도입도 가능 (한국서부발전 단독 적용도 효과 입증 가능).

### C8. "한국전력거래소 (KPX)와의 협업은 가능한가요?"
KPX EPSIS 발전실적 데이터는 본 솔루션 4-source 융합에 이미 포함. KPX와 직접 협업 시 (a) 발전 부하 변동 → 배출 영향 정밀 분석, (b) 전력시장 운영 최적화에 환경 부하 반영 가능. 본 보고서 §Ⅴ 정책 제언에서 별도 항목으로 추가 검토 가능합니다.

### C9. "본 솔루션이 발전사 단가에 미치는 영향은?"
긍정적입니다. (1) 정기 교정 비용 30~50% 절감, (2) EU CBAM 보고 마진 축소로 비용 절감, (3) ESG 등급 향상 → 투자·금융 비용 절감. 5년 누적 ROI 290배 추정 (§Ⅴ B/C 분석). 발전 단가 직접 영향은 미미하지만 영업이익률 + ESG 등급에 기여.

### C10. "한국서부발전이 본 솔루션 IP를 어떻게 처리하나요?"
본 솔루션 코드는 GitHub MIT License로 공개됩니다 (오픈소스). 한국서부발전이 직접 도입 시 라이선스 비용 없음. 본 연구자는 컨설팅 (6개월 무료) + 측정 불확도 자문 제공. 발전사가 자체 개발팀이 코드를 fork하여 운영하는 모델도 가능.

## D 비용·운영 추가 4 (5~8)

### D5. "5년 누적 ROI 290배는 너무 낙관적 아닌가요?"
정확한 지적입니다. ROI 290 = 1호기 기준 (CBAM 261억/년 × 5년 = 1,305억 - 도입 5억 - 5년 OPEX 5억 = 1,295억) / 5억. 가정에 민감합니다: (a) EU CBAM €90/t (보수 가정), (b) 측정 정확도 1%p 개선. 보수 시나리오 (€60/t, 0.5%p 개선)에서도 ROI 50~100배 유지.

### D6. "도입 후 추가 인력 채용 필요한가요?"
최소화 가능합니다. 본 솔루션은 자동화 비율 60% 이상 설계 (P1 auto-execute). 발전사 기존 운영자 1명이 일 30분 추가 작업으로 충분. 다만 도입 초기 6개월은 본 연구자가 컨설턴트로 동행, 그 후 사내 IT팀 1명 분기 1~2일 유지보수.

### D7. "5호기 동시 배포 시 리소스 (서버·네트워크) 부담은?"
경량 솔루션입니다. 5호기 × 5분 간격 SOx/NOx/먼지/CO 4개 = 일 5,760 record × 5 = 28,800 record/day. CPU 1 core + RAM 4GB + 디스크 100GB/년 충분. 본 코드베이스 Docker 컨테이너로 배포 → 수평 확장 가능.

### D8. "유지보수 비용이 도입 비용보다 클 위험은?"
관리 가능합니다. 본 보고서 §Ⅴ B/C 분석에 따라 연 OPEX 1억 = CAPEX 5억의 20%. 5년 누적 OPEX 5억 = 도입 비용과 동일 규모. 다만 (1) 모델 재학습 자동화, (2) Docker 표준화, (3) GitHub Public 외부 감사 가능으로 비용 압박 최소화.

## E 한계·약점 추가 4 (5~8)

### E5. "Gaussian Plume 단순화는 복잡 지형에서 부정확하지 않나요?"
정확합니다. 본 구현은 평탄 지형 가정. 산악·해안 복잡 지형에서는 (1) AERMOD (EPA) 또는 (2) WRF-CMAQ 결합 모델이 더 정확. 다만 (a) 본 §Ⅲ 검증에서 Coverage 0.96 ✓ 도달, (b) 한국서부발전 5개 발전소 (태안·평택·서인천·군산·남제주) 대부분 해안 평탄 지역으로 충분. Phase 2에서 AERMOD 통합 검토.

### E6. "발전소 운영자 신뢰가 정말 4 컴포넌트로 충분히 정량화되나요?"
4 컴포넌트 (측정 + 모델 + XAI + 이력) 외 (a) 통신 신뢰도 (네트워크 지연), (b) 운영자 경험 점수 등 보조 컴포넌트 추가 가능. 본 보고서는 핵심 4개로 단순화. 본 시스템은 컴포넌트 확장 가능 구조 — `src/innovation/operator_trust.py` 4 컴포넌트가 plug-and-play로 추가 가능합니다.

### E7. "Page-Hinkley 변화점 검출의 false positive는 어떻게 관리하나요?"
threshold 파라미터 (delta=0.005, λ=10) 보수적 설정으로 false positive < 5% 달성. 또한 (1) 7일 sliding window로 단기 spike 무시, (2) AirKorea 비교 시 Gaussian Plume 보정 후 잔차 5σ 이상만 트리거. 환경 모니터링 표준 실무와 일관.

### E8. "본 솔루션 도입 후 운영자 직무 변화는 없나요?"
긍정적 방향: (1) 단순 알람 모니터링 → 모델 부적합 케이스 분석 + 정책 결정. 운영자 역량 upskill 기회. (2) OTS 등급 시스템이 책임 범위 명확화. (3) 환경부 보고 신뢰도 향상으로 외부 감독 부담 감소. 다만 도입 초기 3개월 적응 기간 필요 → 운영 매뉴얼 + 교육 프로그램 동반.
